## Pages and routes

This project registers its client-side routes in `src/App.jsx` using React Router. The main pages and their component files are:

- `/` ‚Äî Home (component: `src/home.jsx`)
- `/about` ‚Äî About (component: `src/about.jsx`)
- `/projects` ‚Äî Projects (component: `src/project.jsx`)
- `/contact` ‚Äî Contact (component: `src/contact.jsx`)
- `/journey` ‚Äî Journey (component: `src/journey.jsx`)
- `/resume` ‚Äî Resume (component: `src/resume.jsx`)

The top navigation bar is implemented in `src/navbar.jsx` and styled in `src/navbar.css`. It uses `react-router-dom`'s `<Link>` components for client-side navigation and includes a simple mobile menu toggle.

To add a new page: create a React component in `src/`, then register a route in `src/App.jsx` (see the `createBrowserRouter` usage).

## Featured projects & styling

The home page exposes a small "Featured Projects" area implemented in `src/home.jsx` and styled in `src/home.css`.

- Structure: the projects area is a container with the class `.projects-list`. Each project is represented by an anchor (rendered by `react-router-dom`'s `<Link>` in this project) and an adjacent metadata block (title/description) ‚Äî see `src/home.jsx` for the exact markup.
- Styling: `src/home.css` defines a `.project` rule (card size, border-radius, background settings) and three ID-based selectors (`#project1`, `#project2`, `#project3`) that set background images for each featured card.
- Runtime class assignment: a small JavaScript `useEffect` in `src/home.jsx` selects the rendered anchors inside `.projects-list` and adds the `project` class at runtime. This keeps styling centralized in CSS while still allowing the markup to be generated by React.
- Assets: background images referenced from the project cards live alongside your home styles (the current CSS uses `./images/Project.png`, etc.). Keep those images in `src/images` or `public` depending on your bundling preference.

Notes & accessibility:

- External links: if a project link navigates to an external site (for example a deployed demo on Vercel), prefer using a plain `<a href="..." target="_blank" rel="noopener noreferrer">` so the browser handles external navigation and users get an expected "open in new tab" experience.
- Semantic text: include visible titles and descriptions (as in the example `h4` / `h5` markup) to ensure cards are accessible to screen readers and keyboard users.

